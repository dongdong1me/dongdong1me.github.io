综上：Block实际是将函数及其执行上下文封装起来的对象


Block的变量捕获

使用文初的示例代码，通过 clang -rewrite-objc -fobjc-arc main.m 命令将文件编译成 main.cpp 文件

  • 对于基本数据类型的局部变量，捕获其值
  • 对于对象类型的局部变量，连同其所有权修饰符一同捕获
  • 以指针形式捕获静态局部变量
  • 不捕获全局变量、静态全局变量

__block 修饰符

  • 一般情况下，对被截获变量进行赋值操作需添加__block 修饰符。对于 局部变量非静态变量 进行赋值操作时需要使用 __block 修饰符；而对于 静态局部变量、全局变量、静态全局变量 则不需要使用。
  • __block 修饰符会使其修饰的变量被包装成 __Block_byref_multiplier_0 结构体，其中包含 变量 multiplier 和 __forwarding 指针，通常情况下__forwarding 指针会指向自己。

Block的内存管理

Block的类型

C++ OC 存储区
_NSConcreteGloablBlock NSGlobalBlock 全局数据区
_NSConcreteStackBlock NSStackBlock 栈
_NSConcreteMallocBlock NSMallocBlock 堆

通常情况下Block都是_NSConcreteStackBlock类对象，都设置在栈上，我们通过 clang 命令编译出来的代码中 impl.isa 的类型也为_NSConcreteStackBlock，但实际上并非全是这样，这是因为ARC情况下，在底层block赋值操作时，会调用到_Block_copy，把stack类型的转换成Malloc类型：

  1. 不使用外部变量的block是全局block
  1. 使用外部变量并且未进行copy操作的block是栈block

对栈block copy之后，会创建新的堆block，但是原来的栈block仍然存在，在ARC情况下，随着堆block的作用域结束， 栈block会被销毁，malloc随着引用计数的减少也会被销毁。但是MRC 的情况下，栈block被销毁后，堆block并不会销毁，导致内存溢出。

Block的copy操作

Block类型 源 copy结果
_NSConcreteGloablBlock 全局数据区 什么也不做
_NSConcreteStackBlock 栈 堆
_NSConcreteMallocBlock 堆 增加其引用计数
  • Block 内部没有引用外部变量，Block 在全局区，属于 GlobalBlock
  • Block 内部有外部变量：
    引用全局变量、全局静态变量、局部静态变量：Block 在全局区，属于 GlobalBlock
    引用普通外部变量，用 copy，strong 修饰的 Block 就存放在堆区，属于 MallocBlock；用 weak 修饰的Block 存放在栈区，属于 StackBlock

注意：Block 引用普通外部变量，都是在栈区创建的，只是用 strong、copy 修饰的 Block 会把它从栈区拷贝到堆区一份，而 weak 修饰的 Block 不会；在 ARC 中，用 strong、copy 修饰的 Block，会从栈区拷贝到堆区，所以在 ARC 中，用 strong 修饰和 copy 修饰的 Block 效果是一样的；

block变量在copy时，如果有forwarding存在，那么栈上的forwarding指针会指向堆上的forwarding变量，而堆上的forwarding指针指向其自身，所以，如果对block的修改，实际上是在修改堆上的block变量。这样子就可以保证之后我们都是访问同一个结构体中的变量，这里就是为什么 __block 修饰的变量，在 Block 内部中可以修改的原因了。forwarding指针存在的意义就是，无论在任何内存位置，都可以顺利地访问同一个__block变量。

然后 block 没有截获外部变量、截获全局变量的都是属于全局区的 Block，即 GlobalBlock；其余的都是栈区的 Block，即 StackBlock；只要经过赋值就会被copy到堆上

对于全局区的 Block，是不存在作用域的问题，但是栈区 Block 不同，在作用域结束后就会 pop 出栈，__block 变量也是在栈区的，同理作用域结束也会 pop 出栈。

为了解决作用域的问题，Block 提供了 Copy 函数，将 Block 从栈复制到堆上，在 MRC 环境下需要我们自己调用 Block_copy 函数，这里就是为什么 MRC 下，我们为什么需要用 copy 来修饰 Block 的原因。
